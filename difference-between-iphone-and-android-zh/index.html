iPhone和安卓的体验区别

Posted on 2021-10-02 In 计算机 , 使用

我也是安卓和iOS都用过，并且用过四个品牌的安卓机了，这一篇只是想随便说一下体验上的感受。我的标题里是iPhone和安卓，其实两者并不对等，但是因为iOS系统和iPhone硬件是一体的，安卓会出现在不同的产品上，所以才这么写。

现况

iPhone应该说是先驱，安卓说后起之秀，最初的安卓确实有些问题，但是在几年前安卓和iOS的体验就已经缩小了很多了，大众感知到安卓和iPhone体验缩小又是有几年的延迟了。

系统开放性

iPhone是很封闭的，安卓早期很开放，近年来开放度被手机产商锁紧，但是依然比较开放。

这会带来什么呢，iPhone也曾被比作舒适的监狱，因此早期的破解都称作越狱，苹果允许你做什么你才能做什么，但是给你体验确实不错。

而安卓就更依靠使用者，安卓可以实现很多黑科技，比如早年安卓可以通过一些操作看到微信撤回的消息，但是恶意软件也会利用安卓开放的特点作恶。当然近年来手机产商也在缩紧开放度，但肯定做不到iOS的那种封闭度。

因此安卓的使用上限更高，下限更低，就会对使用者提出更高的要求。

隐私/权限控制

这一方面绝对是iPhone做的好很多，得益于其封闭性。虽然现在安卓机也在推进隐私保护这一块，虽然受限于安卓本身特性，但是体验上就是始终不如iPhone。

后台管理

这一点就是有些仁者见仁了。安卓的后台就是大多数人理解的后台，是程序真正在运行。

而iOS的后台是“假”后台，iPhone只允许程序介入自己提供的少数API（Application Programming Interface）来保持在后台运行。一般的iOS程序在退出后就不再运行，只是会把当前状态存入内存中，再打开就会很快，营造一种真后台的感觉。因此才有说iPhone清后台只会让机器变慢的说法。

因此在此基础上两者就会在体验上出现不同的区别，比如：

  • 安卓因为是真的后台，程序间的切换更加无缝丝滑。当然两者的差距本身其实也就比较小。
  • iPhone因为没有真后台，在使用比如微信这样的软件时，在你打开微信后，该app才启动，然后去连接腾讯服务器获取消息，消息获取就需要等一会。而安卓的因为后台在运行，已经就把消息拉下来了，打开就有最新消息，体验上就更好。
  • 但也正是因为安卓允许真后台，所以恶意程序也能利用这个后台机制，常驻后台完成自己的恶意目的，正如一开始所说，就要求用户去甄别和限制这样的程序。
  • 如果安卓app的开发者不合格，也会导致程序在后台占用过多资源的情况。

推送机制

经常有吐槽的iPhone推送延迟的，这就涉及推送机制。

用微信举例：

  • iPhone的推送机制是：腾讯 -> 苹果 -> iPhone
  • 国外带GMS的安卓机的推送机制是：腾讯 -> 谷歌 -> 手机
  • 国内的安卓机的推送机制是：腾讯 -> 手机

这里先科普一下GMS（Google Mobile Service），GMS是国外安卓必备的一个组件。国内的安卓与国际实际上是割裂的，因为没有GMS。

iPhone和国际带GMS的手机，都是第三方把推送给到苹果/谷歌，然后苹果/谷歌把推送合到一起，再推给用户。这用户的手机只需要和苹果/谷歌建立连接即可，省电省内存。当然如果苹果/谷歌的服务器抽风了就会遇到推送延迟。

然而国内安卓因为没有GMS，都是第三方直接推给用户，这样就要求用户和每一个推送服务都建立连接，费电费内存。国内安卓内存大也是很大程度因为这个，因为国内安卓机需要和很多第三方同时保持连接。

一般来说，还是iPhone/GMS的方式被认为更好，但是确实苹果服务器偶尔抽风体验不太好，但总体来说，苹果的推送机制还是要更好一些，虽然用户并感知不到。

硬件配置

其实写这一节主要是因为对于安卓和iPhone的了解可能会滞后的情况，实际上同价位上现在安卓机和iPhone的配置已经差不多了。iPhone的CPU确实很强，但是安卓也在一些方面比iPhone好。同一年的的手机配置都差不多的，新一年的iPhone比老一年的安卓配置好，同样新一年的安卓也要比老一年的iPhone配置好。

总结

总的来说，其实两个系统现在体验上还是有区别，但是一个能完成的事另一个基本都能完成，最好还是两个都试用一下选一个自己用的舒服的。
